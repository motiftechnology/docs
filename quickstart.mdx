## Prerequisites

Before you begin, ensure you have:

- **Node.js** >= 20
- **Go** >= 1.21
- **PostgreSQL** database
- **Redis** (optional, for multi-node deployments)
- **Git**

## Backend API

The NestJS backend provides the REST API for managing tenants, conversations, messages, and more.

### Installation

```bash
# Clone the repository
git clone https://github.com/motiftechnology/chatAfika.git
cd chatAfrika

# Install dependencies
npm install
```

### Environment Setup

Create a `.env` file in the root directory:

```env
# Database
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_password
DB_NAME=chaterafrika

# JWT Secrets
ACCESS_SECRET=your_access_secret
REFRESH_SECRET=your_refresh_secret
SDK_TOKEN_EXPIRY=15m

# App Configuration
APP_NAME=ChaterAfrika
APP_PORT=3000
APP_ENV=development
API_URL=http://localhost:3000

# Redis (optional)
REDIS_HOST=localhost
REDIS_PORT=6379
```

### Database Setup

```bash
# Run migrations
npm run migration:run

# Run seeders (optional)
yarn seed:run
```

### Start the Server

```bash
# Development mode
npm run start:dev

# Production mode
npm run start:prod
```

The API will be available at `http://localhost:3000`

<Note>
Check the [Backend API documentation](/guides/backend/overview) for detailed information.
</Note>

## Realtime Service

The Go-based WebSocket service handles real-time messaging, presence, and typing indicators.

### Installation

```bash
# Clone the repository
git clone https://github.com/motiftechnology/chatAfirka-realtime.git
cd chatAfrika-realtime

# Install dependencies
go mod download
```

### Environment Setup

Create a `.env` file:

```env
# Server
SERVER_PORT=8080
SERVER_HOST=localhost

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

# JWT
JWT_SECRET=your_jwt_secret
JWT_ISSUER=chaterafrika
JWT_AUDIENCE=chaterafrika-sdk

# Environment
ENV=development
```

### Start the Service

```bash
# Development mode
go run cmd/server/main.go

# Or build and run
make build
./server
```

The WebSocket service will be available at `ws://localhost:8080/ws`

<Note>
Check the [Realtime Service documentation](/guides/realtime/overview) for detailed information.
</Note>

## JavaScript SDK

The headless SDK for integrating ChaterAfrika into your applications.

### Installation

```bash
npm install @chatafrika/sdk
```

### Basic Usage

```typescript
import { ChatAfrika } from '@chatafrika/sdk';

// Initialize the SDK
const client = new ChatAfrika({
  apiUrl: 'http://localhost:3000',
  wsUrl: 'ws://localhost:8080',
  sdkToken: 'your_sdk_token'
});

// Connect to realtime
await client.connect();

// Join a conversation
await client.conversations.join('conversation-id');

// Send a message
await client.messages.send({
  conversationId: 'conversation-id',
  content: 'Hello, world!'
});

// Listen for messages
client.messages.on('message', (message) => {
  console.log('New message:', message);
});
```

<Note>
Check the [SDK documentation](/guides/sdk/overview) for detailed information and examples.
</Note>

## Verify Installation

### Test Backend API

```bash
# Health check
curl http://localhost:3000/health

# Register a new user
curl -X POST http://localhost:3000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "SecurePassword123!",
    "company_name": "Test Company"
  }'
```

### Test Realtime Service

```bash
# Using wscat
npm install -g wscat

wscat -c "ws://localhost:8080/ws" \
  -H "Authorization: Bearer YOUR_SDK_TOKEN"
```

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Database connection errors">
    Ensure PostgreSQL is running and the connection details in `.env` are correct.
  </Accordion>
  <Accordion title="WebSocket connection fails">
    Verify the realtime service is running and the WebSocket URL is correct.
  </Accordion>
  <Accordion title="Port already in use">
    Change the port in your `.env` file or stop the service using that port.
  </Accordion>
</AccordionGroup>

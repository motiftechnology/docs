## Sending Messages

<Warning>
**Messages are sent exclusively over WebSockets.**

This guarantees ordering, real-time delivery, typing indicators, presence, and receipts.

Use the JavaScript SDK to send messages via WebSocket: `client.conversations.get(conversationId).sendMessage(content)`

REST APIs are used for history, analytics, and internal automation only.
</Warning>

## Welcome

This API reference documents the REST API endpoints available in ChaterAfrika. The REST API is used for:

- **Message History** - Retrieving past messages and conversations
- **Analytics** - Accessing metrics, reports, and bot analytics
- **Internal Automation** - Managing resources, configurations, and administrative tasks
- **Data Management** - Creating, updating, and querying conversations, users, and other entities

For sending messages in real-time, use the [WebSocket connection](/quickstart#websocket-connection) or the [JavaScript SDK](/quickstart#javascript-sdk).

## Authentication

All API endpoints are authenticated using Bearer tokens. Include your access token in the Authorization header:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```
